🤖 AI models — master kill-switch

Disable every model the picker can offer in one click. Users already mid-conversation finish their current turn; new ``/start`` flows that try to pick a model fall through to the "no models available" path. Re-enable wipes the entire disabled_models table.

💳 Payment gateways — master kill-switch

Disable every payment gateway and crypto ticker. Existing pending invoices are unaffected (they finalize via IPN as normal); new top-up attempts hit a "no gateway available" path. Useful when Zarinpal / TetraPay are down or NowPayments IPNs are being forged.

Currently disabled: {{ signals.disabled_gateways_count }} of {{ signals.total_gateways_count }} gateway key(s).

🛑 Force-stop bot

Send SIGTERM to PID {{ signals.pid }}. The bot's asyncio loop will unwind cleanly: cancel background tasks, close the DB pool, close the Telegram session, exit. Your process supervisor (systemd / docker / pm2) will restart the bot automatically — this is your one-click "kill it before it bleeds out" button. Do not use this if you only want to pause traffic; use the kill-switches above instead.

The web admin will become unreachable for ~5–15 seconds while the bot restarts. Refresh the page after that to confirm the new PID.
