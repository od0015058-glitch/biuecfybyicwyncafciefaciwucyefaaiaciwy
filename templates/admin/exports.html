Notes for incident-response & quarterly-close exports
  • Every export is recorded in the audit log with row count and filters under the slugs transactions_export_csv, monetization_export_csv, system_usage_export_csv, and admin_audit_export_csv so a forensic timeline can answer "did anyone pull this data on Tuesday?".
  • All four exporters defang CSV-formula injection (=, +, @, \t, \r as the leading character) by prepending a TAB so a malicious refund-reason like =HYPERLINK("https://attacker", "click me") can't execute when the operator opens the CSV in Excel. Negative dollar amounts (-) are intentionally not defanged because false-flagging them would mangle every legitimate accounting CSV; the residual narrow attack vector is documented in web_admin._csv_quote.
  • For exports larger than the per-endpoint row cap, slice the timeline with since / until and concatenate offline. The DB-side keyset pagination keeps cost constant per batch, so a quarter-of-a-billion-row table and a 100-row table cost the same per fetched row.
