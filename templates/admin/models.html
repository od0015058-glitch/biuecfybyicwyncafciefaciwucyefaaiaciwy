Per-gateway model exceptions

Block a specific model × gateway pair without disabling either parent globally. When a user with the matching active model picks the matching payment method, the bot refuses with "this payment method isn't available for your current model" and the user is steered to a different model or a different gateway. Useful for policy carve-outs (e.g. expensive vision models that shouldn't be funded via card-only gateways) without taking the model or the gateway off the menu for everyone else.
