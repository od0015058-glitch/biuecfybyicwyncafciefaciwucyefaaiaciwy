Admin roles

DB-tracked admin roles graduate the bot's admin surface beyond the legacy single-password env list. Each role grants a different slice of the Telegram-side admin commands (and, in a future PR, the web panel itself).
  • viewer — read-only. /admin_metrics, /admin_balance, this dashboard.
  • operator — viewer + /admin_broadcast, promo / gift code CRUD.
  • super — operator + /admin_credit, /admin_debit, refunds, user-field edits. Equivalent to a legacy ADMIN_USER_IDS entry.
Telegram ids in ADMIN_USER_IDS remain super by virtue of the env-list backward-compat fallback even without a row here.
